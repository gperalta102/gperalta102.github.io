Dynamic Video Streaming Website

Video streaming website created for senior level software engineering class.

Dynamic video streaming website created utilizing and AWS EC2 instance running a nginx webserver and a Node.js runtime environment. Nginx take valid request and passes them on to the Express app. The Express app performs proper routing of requests, handles interactions with database, and responds with a dynamically generated html page created utilizing the express framework and pug.js templating engine.

Development Style

The final project for my undergraduate Software Engineering class. We were required to create a dynamic video streaming website from scratch. We were requried to implement agile software development. This consisted of daily meetings between the team to discuss what we have contributed to the project and what we are working on currently or moving on to. Once a week we met with the professor who was portraying the customer.

Back-End

Beginning with a AWS EC2 t2.micro instance running CentOS linux. We used a MariaDB SQL database to store the user information, their watchlist etc. While this is not maintainable, due to the time constraints of the project we had to make compromises. The MariaDB database was chosen due to SQL being the laguage taught at the school for database systems. This allowed us to begin producing a project sooner since most of us had experience with mySQL. NGINX was chosen as the webserver due to being faster than apache at this scale.

Trying to keep the project for the majority of the team members we decided to go with the Node.js runtime environment with and Express. Working in one langauage allowed for faster upskilling, greater team flexibility, and more effective intra-team knowledge sharing. This allowed for everyone in the group to work as full-stack.

Front-End

Express allows the use of template engines. This allows you to specify the structure of an output in a template. These templates were used to generate HTML dynamically. Using the template engine, we are able to reuse code for multiple pages. The template has place-holder data that is replaced with the proper information from the database Express generated the HTML pages. The templating engine used was Pug.js(Jade).

I was responsible for the front-end design and implementation. We wanted to plan around the user experience of the website.

User Experience

More and more users watch videos on their phones and tablets. With this in mind, the decision was made to go with a mobile first design. With this in mind, we decided to go with Bootstrap 4 framework. Bootstrap 4 is mobile first. Bootstrap 4 allowed us to theme the website faster and create a responsive site that can scale to a full browser experience but also scale down to a mobile phone.

User Interface

Using popular website as reference, we created a UI that was usable as on phones but still looked and worked great on desktop. The UI consisted of large buttons, sidebar and a static navigation bar. This was the most common design for many streaming website.

Page Design

To design the UI of the website, I utilized Microsoft Expression Design 4 to create mock-up of what the pages are supposed to look. Bootstrap is designed with the idea of 12 columns and rows. Keeping this in mind allowed me to design pages that looked good on desktop but with the knowledge that the page would look great on mobile as well.
